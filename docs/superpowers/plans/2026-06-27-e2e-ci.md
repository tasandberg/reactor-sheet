# E2E CI Implementation Plan

> Executed directly (goal: ship a PR and drive CI green). Spec:
> `docs/superpowers/specs/2026-06-27-e2e-ci-design.md`.

**Goal:** Boot Foundry v14 + OSE + reactor-sheet in GitHub Actions and run
`@playwright/test` user-click specs against the character sheet's core flows.

**Architecture:** Reuse the ose-foundry-core `quench-tests-and-ci` Foundry-boot
harness (felddy docker, EULA auto-accept via `activate-world.mjs`, license-secret
workflow) but assemble a different `/data` (OSE pinned release + built reactor-sheet
module) and replace the Quench runner with Playwright specs that drive real DOM.

**Tech stack:** felddy/foundryvtt:14 · Playwright (`@playwright/test`) · bash/jq ·
GitHub Actions.

## Global constraints

- Foundry **v14** only (`felddy/foundryvtt:14`).
- OSE system = NecroticGnome release **2.2.2** zip; rewrite `system.json`
  `compatibility.verified` → `"14"` (CI masquerade — no v14-verified OSE exists).
- reactor-sheet module id `reactor-sheet`; files needed in the module dir:
  `module.json`, `dist/`, `lang/`. Built via `pnpm build` before data-dir assembly.
- Sheet registered as `ose.ReactorSheet`, `makeDefault: true` for `character`/`npc`.
- Never push to main; work on branch `e2e-ci`, open a PR.
- E2E deps isolated in `tools/e2e/package.json` (keep Playwright out of app deps).

## File structure

```
tools/e2e/
  package.json            # @playwright/test + playwright (pinned)
  playwright.config.ts    # testDir=specs, globalSetup, baseURL, single worker
  global-setup.ts         # join GM, seed "E2E Fighter" actor + items via API
  helpers.ts              # joinAsGM(page), openCharacterSheet(page), newChatCount
  setup-data-dir.sh       # assemble /data: OSE 2.2.2 + reactor-sheet module + world
  activate-world.mjs      # EULA accept + world-active poll (reused ~verbatim, v14)
  fixtures/world/world.json
  specs/
    smoke.spec.ts         # sheet opens, .reactor-sheet root + tablist
    tabs.spec.ts          # tab nav switches panels
    ability.spec.ts       # STR check → chat message
    save.spec.ts          # save → chat message
    attack.spec.ts        # weapon attack (mode toggle + roll) → chat
    equip.spec.ts         # equip toggle flips state
    coin.spec.ts          # coin edit persists to item
  .gitignore
.github/workflows/e2e.yml
```

Plus `data-testid` additions in:
`TabBar.tsx`, `AbilityPlaques.tsx`, `SavesExploration.tsx`, `AttacksTable.tsx`,
`InventoryViewDnd.tsx` (RowEquip), `WealthSection.tsx`.

## Tasks

1. **Vendor harness** — copy `activate-world.mjs` verbatim; write reactor-sheet
   `setup-data-dir.sh` (download OSE 2.2.2 zip → `/data/Data/systems/ose`, rewrite
   verified→14; copy `module.json`+`dist`+`lang` → `/data/Data/modules/reactor-sheet`;
   world fixture). Deliverable: `setup-data-dir.sh /tmp/d <repo>` assembles a valid dir.
2. **Playwright scaffold** — `package.json`, `playwright.config.ts`, `helpers.ts`,
   `global-setup.ts` (seed actor: character + weapon `Dagger`, armor `Leather`,
   treasure `Gold piece` qty 50; set `core.sheetClass=ose.ReactorSheet`). `smoke.spec.ts`.
3. **testids** — add `data-testid` to the 6 components (tabs, abilities, saves,
   attack, equip, coin) using the maps from the spec.
4. **Specs** — one per flow (tabs, ability, save, attack, equip, coin), resilient
   waits (`expect.poll` on chat count / state), single worker, serial.
5. **Workflow** — `.github/workflows/e2e.yml` adapted from `quench.yml`: pnpm build,
   assemble data dir, boot felddy:14, EULA, restart, await, `npx playwright test`.
6. **PR + iterate** — push branch, open PR, watch Actions, fix until green.

## Seed actor shapes (OSE)

```js
// in global-setup, page.evaluate as GM:
const actor = await Actor.create({ name: "E2E Fighter", type: "character" });
await actor.createEmbeddedDocuments("Item", [
  { name: "Dagger", type: "weapon",
    system: { damage: "1d4", melee: true, missile: true, equipped: false, qty: { value: 1 } } },
  { name: "Leather Armor", type: "armor",
    system: { equipped: false } },
  { name: "Gold piece", type: "item",
    system: { treasure: true, quantity: { value: 50 } } },
]);
await actor.setFlag("core", "sheetClass", "ose.ReactorSheet");
```
(Exact OSE field names validated live in the smoke run; adjust if the data model
differs — this is the first thing the CI smoke test surfaces.)

## Risks / iterate-live

- OSE 2.2.2 data model under v14 — smoke test confirms render.
- Roll dialogs: `rollCheck`/`rollSave` may open a modifier dialog; specs handle a
  confirm button if present, then assert chat.
- v14 chat DOM selector (`.chat-message`) — confirm in first run.
- Repo secrets `FOUNDRY_USERNAME`/`FOUNDRY_PASSWORD` must exist or CI fails fast.
